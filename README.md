# TIORA Landing Page — SMD‑расстановщик

Одностраничный сайт (landing page / SPA) для B2B‑аудитории: инженеры, производственные команды, отделы закупок. Проект реализован как статическая страница с секциями о продукте, преимуществах, характеристиках и формах заявки. Цель репозитория — быстро запустить сайт, заменить контент под свою компанию и без лишней настройки вывести в продакшн.

## Демонстрация и просмотр
Для подробной инструкции по просмотру сайта см. отдельный документ: `VIEWING_GUIDE.md`.

Быстрый вариант:

```bash
python -m http.server 8000
```

Откройте `http://localhost:8000`.

## Стек технологий
- HTML5
- CSS3
- Vanilla JavaScript
- SVG‑ассеты

> Note:
> В проекте нет зависимости от Node.js. Это статическая страница, которая может быть открыта напрямую или через локальный сервер.

## Архитектура и структура проекта
Фактическая структура репозитория:

```
.

├── design-system.md        # UI-гайд и визуальные принципы
```

## Контент и кастомизация
Самый важный раздел. Все тексты находятся в `index.html`, а полный текстовый контент — в `landing-content.md`.

### Изменить название продукта, слоган, CTA в Hero
Файл: `index.html` → секция `#hero`.

Что менять:
- Заголовок `<h1>`
- Подзаголовок `<p class="lead">`
- Тексты CTA-кнопок
- KPI‑строки

Пример:

```html
<h1>Настольный SMD‑расстановщик для серийной сборки</h1>
<p class="lead">Инженерная точность и сервис уровня enterprise.</p>
<a class="btn btn-primary" href="#contact">Запросить демонстрацию</a>
```

Важно не ломать:
- `id="hero"` и структуру классов — они используются якорной навигацией и стилями.

### Изменить блок «О продукте»
В текущей версии роль «О продукте» выполняют секции:
- `#benefits` (Почему TIORA)
- `#specs` (Технические характеристики)

Если нужен отдельный блок «О продукте», добавьте новый `<section>` между Hero и Сегментами и включите его в навигацию.

### Обновить преимущества
Файл: `index.html` → секция `#benefits`.

Что менять:
- Каждая карточка в `<div class="benefit">`.

Пример:

```html
<div class="benefit">
  <h3>Процессный подход</h3>
  <p>Профили, библиотека компонентов, контроль качества.</p>
</div>
```

Важно не ломать:
- структуру `.benefits` и `.benefit` — это влияет на сетку.

### Технические характеристики
Файл: `index.html` → секция `#specs`, таблица `.specs-table`.

Что менять:
- значения в ячейках таблицы.

Пример:

```html
<tr><td>Производительность</td><td>8 000 cph</td></tr>
```

Важно:
- сохраняйте единицы измерения и единый формат значений.

### Замена изображений
Папка: `assets/`.

Что менять:
- `machine-placeholder.svg` → заменить на фото/рендер оборудования.
- `demo.mp4` / `demo.webm` → заменить на реальное демо‑видео.
- `demo-placeholder.svg` → превью (если видео нет).

Важно:
- сохраняйте имена файлов или обновляйте пути в `index.html`.

### Контакты и реквизиты
Файл: `index.html` → Header и Footer.

Что менять:
- Телефон (и `href="tel:"`)
- Email
- Реквизиты

Пример:

```html
<a href="tel:88000888888">8 800 088 88 88</a>
<p>Email: sales@tiora.ru</p>
<p>Реквизиты: [уточнить]</p>
```

Важно:
- Согласовать юридические формулировки с юристом.

### Настройка формы заявки
Файл: `index.html` (формы) и `script.js` (валидация).

Что менять:
- Состав полей (HTML)
- Отправку (JS)

Важно:
- Не удаляйте `required` у обязательных полей.
- Сохраняйте `data-form` для логики обработки.

### Добавить новый блок/секцию
Пошагово:
1. Добавьте новый `<section>` в `index.html`.
2. Пропишите `id` для якоря.
3. Добавьте пункт меню в навигацию (header).
4. При необходимости — стили в `styles.css`.

### Изменить цветовую палитру и шрифты
Файл: `styles.css` → блок `:root`.

Что менять:
- `--accent`, `--bg`, `--text` и другие токены.
- Подключение шрифтов в `<head>` (`index.html`).

### Изменить ссылки в футере и навигации
Файл: `index.html`.

Важно:
- Сохраняйте корректные `href="#section-id"`.

### Локализация/язык
В текущей версии язык один (`lang="ru"`).

> Note:
> Для двуязычной версии рекомендуется вынести тексты в отдельные JSON‑файлы и подключать их через JS или шаблонизатор.

## Формы и обработка заявок
Формы сейчас работают как клиентская заглушка (валидация + success‑сообщение).

Для подключения:
1. Добавьте эндпоинт API.
2. В `script.js` замените обработчик `submit` на `fetch`.

Пример (фрагмент):

```js
const endpoint = 'https://api.example.com/lead';
await fetch(endpoint, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload)
});
```

> Warning:
> При подключении форм обязательно добавьте серверную валидацию и антиспам (например, honeypot или reCAPTCHA).

## SEO и мета-информация
Файл: `index.html` → `<head>`.

Что менять:
- `<title>`
- `<meta name="description">`
- Open Graph и Twitter Card (добавить при необходимости)

Пример OG‑тегов:

```html
<meta property="og:title" content="TIORA — SMD‑расстановщик" />
<meta property="og:description" content="Инженерная точность и сервис для B2B." />
<meta property="og:image" content="assets/og-image.jpg" />
```

Также рекомендуется добавить:
- `favicon.ico` или PNG‑иконки
- `robots.txt` и `sitemap.xml` при размещении на домене

## Деплой
### Вариант 1: Netlify (static)
1. Скопируйте весь каталог проекта.
2. Загрузите в Netlify как Static Site.
3. Build command не нужен, publish directory — корень проекта.

### Вариант 2: Nginx
1. Скопируйте файлы в `/var/www/tiora`.
2. Пример конфига:

```nginx
server {
  listen 80;
  server_name example.com;
  root /var/www/tiora;
  index index.html;
}
```

> Note:
> Проверьте MIME‑типы для `.webm` и `.svg`.

## Troubleshooting
- Видео не воспроизводится: проверьте, что `muted` включён и файлы доступны.
- Шрифты не грузятся: проверьте доступ к Google Fonts или замените на локальные.
- Формы не отправляются: убедитесь, что endpoint доступен и CORS настроен.
- Сайт не открывается локально: убедитесь, что сервер запущен и порт свободен.
