# Viewing Guide — как посмотреть сайт

Этот документ описывает все способы просмотра и проверки лендинга TIORA. Подходит для дизайнеров, менеджеров, инженеров и QA. Основано на текущей статической реализации проекта.

## 1. Самый быстрый способ (рекомендуется)
### Запуск локального сервера через Python
1. Откройте терминал в корне проекта `d:\сайт 2`.
2. Выполните команду:

```bash
python -m http.server 8000
```

3. Откройте браузер и перейдите на:

```
http://localhost:8000
```

4. Для остановки сервера нажмите `Ctrl + C` в терминале.

> Note:
> Этот способ корректно обслуживает видео, шрифты и загрузку ассетов.

### Если порт 8000 занят
Используйте другой порт:

```bash
python -m http.server 8080
```

## 2. Альтернативный способ (если есть Node.js)
> Предположение: Node.js установлен.

```bash
npx serve .
```

После запуска откройте адрес, который покажет `serve` (обычно `http://localhost:3000`).

## 3. Просмотр через VS Code Live Server
1. Откройте папку проекта в VS Code.
2. Установите расширение Live Server.
3. Откройте `index.html`.
4. Нажмите `Go Live`.

## 4. Прямое открытие файла (самый простой, но не идеальный)
Откройте `index.html` двойным кликом. Возможные ограничения:
- шрифты могут не загружаться,
- видео может быть заблокировано браузером,
- CORS‑политики могут блокировать внешние ресурсы.

Используйте этот вариант только для быстрой проверки текста и структуры.

## 5. Просмотр на другом устройстве в локальной сети
1. Запустите сервер на компьютере как в разделе 1.
2. Узнайте IP‑адрес компьютера (Windows):

```bash
ipconfig
```

3. На другом устройстве откройте:

```
http://<ваш_ip>:8000
```

> Warning:
> Убедитесь, что локальный фаервол не блокирует входящие подключения.

## 6. Что проверить при просмотре (контрольный список)
### Контент и структура
- Все секции отображаются в правильном порядке.
- Тексты соответствуют B2B‑стилю.
- Заголовки и CTA читаемы и логичны.

### Формы
- Все обязательные поля помечены.
- Нельзя отправить форму без заполнения обязательных полей.
- Показываются сообщения об ошибках и успехе.

### Видео
- Видео запускается автоматически (muted + autoplay).
- Модальное окно открывается по кнопке play.
- Закрытие по крестику, клику на фон и ESC.

### Визуальный стиль
- Контраст текста достаточный.
- Карточки выровнены и не «ломают» сетку.
- Шрифты загрузились корректно.

### Адаптивность
- Проверить ширины 360px, 768px, 1024px, 1440px.
- На мобильных: навигация скрыта, CTA и телефон доступны.

## 7. Проверка в разных браузерах
Рекомендуется проверить:
- Chrome
- Edge
- Firefox

Особое внимание:
- Автозапуск видео
- Скролл‑поведение якорей
- Стили форм и фокус‑состояния

## 8. Как заменить тестовое видео на своё
1. Подготовьте файлы:
   - `demo.mp4`
   - `demo.webm` (опционально, но желательно)
2. Замените файлы в `assets/`.
3. Обновите страницу.

Если имена файлов другие — измените пути в `index.html`:

```html
<source src="assets/ВАШ_ФАЙЛ.mp4" type="video/mp4" />
```

## 9. Что делать, если видео не воспроизводится
- Убедитесь, что `muted` включен (иначе autoplay блокируется).
- Убедитесь, что `assets/demo.mp4` действительно существует.
- Проверьте MIME‑типы на сервере (особенно для `.webm`).

## 10. Проверка перед публикацией
- Все `[уточнить]` заменены реальными значениями.
- Контактные данные актуальны.
- Видео и изображения заменены на реальные.
- Формы подключены к серверу (если требуется).
- Добавлены favicon, OG‑мета, robots/sitemap (для SEO).
